## C.1 AIClient — cache, retry, observabilidade

Arquivo: `backend/app/services/ai/ai_client.py` (158 linhas).

| Aspecto | Estado | Risco | AUD |
|---|---|---|---|
| Cache key | `sha256(text.lower().strip())[:24]` (96 bits) | 🟢 colisão desprezível | — |
| Cache TTL | 7 dias (`7 * 24 * 3600`) | 🟢 OK | — |
| Cache scope | prefixo único `ai:` global | 🟢 (visão hoje não cacheia) | — |
| Retry trigger | `if "429" in str(exc)` | 🟡 frágil — depende de `str(exc)`; deveria filtrar `groq.RateLimitError` ou `httpx.HTTPStatusError` com `.response.status_code == 429` | AUD-012 |
| Tentativas | 4 com backoff 15s/30s/60s/120s = **225s acumulados** | 🟡 timeout uvicorn keep-alive 5s, sem timeout global; Caddy default proxy_read_timeout pode cortar antes | AUD-012 |
| Vision retry | Mesma lógica, sem cache | OK estrutural | — |
| Logs de tokens | `logger.info("Groq tokens — entrada: %d, saída: %d", …)` sem user_id, request_id, modelo, custo | 🟡 sem dimensão para agregação (custo por usuário, modelo, endpoint) | AUD-013 |
| Conexão Redis | `async with aioredis.from_url(REDIS_URL, decode_responses=True)` em `_get_cached`/`_set_cached` — abre+fecha conexão por chamada | 🟡 latência adicional + pressure no socket pool | AUD-014 |
| Singleton | `get_ai_client()` global | 🟢 OK | — |
| Modelos | constantes hardcoded `_TEXT_MODEL`/`_VISION_MODEL` | OK por enquanto | — |

## C.2 Duplicação MealParser ↔ VisionParser

Comando: `diff -u meal_parser.py vision_parser.py` (artefato `C2-parsers-diff.txt`) + script de comparação por método.

**Métodos em comum** (mesmas assinaturas):

| Método | Linhas (meal vs vision) | Igualdade funcional | Observação |
|---|---|---|---|
| `__init__` | igual | 100% | trivial |
| `_identify_foods` | 17 vs 16 | ~12% (zip) | **Legitimamente diferente** — entrada texto vs imagem; aceitável |
| `_lookup_and_fill` | 82 vs 87 | funcionalmente equivalente; difere em formatação interna | **Duplicação real** |
| `_estimate_macros_batch` | 37 vs 37 | **idêntico exceto 1 palavra na docstring** | **Duplicação real (100%)** |

**Métodos exclusivos**

- `MealParser.parse(...)` (entry point texto)
- `VisionParser.parse_base64(...)` (entry point imagem)

**Quantificação**: 147 linhas diferem entre os 2 arquivos (de 553 totais ≈ 27%). A maior parte da divergência é em `_identify_foods` e nos entry points; o resto é duplicação evitável.

**Plano de extração**

